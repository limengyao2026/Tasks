# 阶段四任务编后感
## 一、任务完成情况
本次任务完成了 Geek 组旧版招新网站的基础复刻，实现了以下核心功能：
1.  还原了网站的整体结构：导航栏、About Us 卡片模块、Contact Us 区块、页脚
2.  完成了关键交互功能：
    - 紫色边框卡片+鼠标悬停放大效果
    - E-mail按钮点击复制邮箱并弹出提示框
    - GitHub按钮跳转到B站视频的彩蛋效果
3.  成功将网页部署到 GitHub Pages，可通过公开链接访问

## 二、遇到的问题与反思
本次任务中，我最大的遗憾是没有成功把手绘的图片加载到页面上。我花了一天半的时间准备和处理图片，但由于路径配置问题，图片始终无法正常显示，只能以文字占位替代。这让我深刻意识到：
- 图片路径是前端开发中非常基础但关键的细节，哪怕是一个小小的文件名错误、中文路径，都会导致图片加载失败
- 遇到问题时，应该及时拆解排查（比如用绝对路径测试、用浏览器开发者工具看报错），而不是盲目反复上传，浪费了大量时间
- 代码和资源文件的管理习惯很重要，以后我会养成用英文命名、统一存放在`assets`文件夹的习惯，避免类似问题

## 三、学习收获
虽然图片没有成功显示，但这次任务让我对前端开发有了更完整的实践理解：
1.  学会了HTML、CSS和基础JavaScript的结合使用，实现了交互效果
2.  掌握了GitHub Pages的部署流程，把本地代码变成了可公开访问的网页
3.  体会到了“从设计到实现”的差距，也明白了调试和问题排查的重要性
4.  更重要的是，我在这个过程中没有放弃，即使遇到了棘手的图片问题，也坚持完成了其他所有功能，这种解决问题的韧性对我来说是更大的收获
