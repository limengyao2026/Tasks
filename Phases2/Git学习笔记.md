# Git 基础提交流程学习笔记
## 一、核心命令说明
| 命令 | 作用 | 关键说明 |
|------|------|----------|
| `git clone` | 克隆远程仓库到本地 | 首次操作必须执行，将远程仓库完整复制到本地 |
| `git add` | 将文件添加到暂存区 | 把修改的文件从工作区提交到暂存区，准备提交 |
| `git commit` | 将暂存区文件提交到本地仓库 | 生成版本快照，必须填写提交说明 |
| `git push` | 将本地仓库推送到远程仓库 | 把本地提交同步到GitHub远程仓库，完成代码共享 |

## 二、完整流程学习
1.  **克隆仓库**：
- 作用：`git clone` 会在本地创建一个和远程完全一致的仓库，包含所有历史版本，分支信息
- 场景：第一次获取远程仓库的代码，比如自己新建仓库后首次拉到本地
- eg:git clone https://github.com/用户名/仓库名.git
2.  **创建文件**：
- 操作：在本地仓库目录中新建`hello.md`文件，写入`# Hello 你的姓名`，属于**工作区**操作，在工作区完成
- eg:新建hello.nd,写入内容，Git会检测到文件的修改，新建
3.  **添加暂存**：
- 作用:`git add` 把文件从工作区移动到暂存区，相当于“打包待提交”
- 细节：
- git add 文件名 ：只添加指定文件的修改
- git add . ：添加当前目录下所有修改、新建的文件（常用）
​- git add -u ：只添加已被跟踪（曾经提交过）的文件的修改，不包含新建文件
4.  **本地提交**：
- 作用:`git commit` 把暂存区的文件永久保存到本地仓库，生成版本记录
- 要求:必须填写提交说明，且说明要清晰、规范，推荐使用约定式提交（Conventional Commits）格式，比如：feat: 新增hello.md文件 （feat表示新功能）,fix: 修复hello.md中的拼写错误 （fix表示修复bug）,docs: 更新README.md文档 （docs表示文档修改）
​
5.  **远程推送**：`git push` 把本地的版本记录同步到GitHub远程仓库，完成提交

## 三、注意事项
- 提交说明要清晰，推荐使用`feat:`（新功能）、`fix:`（修复）、`docs:`（文档）等前缀
- 推送前建议先执行`git pull`拉取远程最新代码，避免冲突
- 注意空格问题，在"以及<后要空格，不然会报错

## 四、hello.md完整提交流程展示
### 前置准备
 1.  安装 Git，配置全局用户名和邮箱
     ```bash
     git config --global user.name "limengyao2026"
     git config --global user.email "2058825803@qq.com"
     ```
 2.  克隆远程仓库到本地桌面
     ```bash
     git clone https://github.com/limengyao2026/Tasks.git
     ```
 3.  进入本地仓库目录
     ```bash
     cd Tasks
     ```
 ### 完整提交流程
 1.  **创建文件**：新建 `hello.md`，写入内容 `# Hello 李孟瑶`
     ```bash
     echo "# Hello 李孟瑶" > hello.md
     ```
 2.  **添加到暂存区**：将文件交给 Git 管理
     ```bash
     git add hello.md
     ```
     > 说明：出现 `LF will be replaced by CRLF` 是 Windows 换行符自动转换警告，不影响提交
 3.  **提交到本地仓库**：添加提交备注，记录本次修改
     ```bash
     git commit -m "feat: 提交hello.md，完成首次Git提交"
     ```
 4.  **推送到远程仓库**：将本地修改同步到 GitHub
     ```bash
     git push origin main
     ```
     > 说明：首次推送需通过浏览器授权 GitHub 账号，授权后 Git 会缓存凭证，后续推送免登录
     
## 五、Git Bash 提交体验与更高效的方式
### 1. Git Bash 提交是否麻烦？
对于新手来说，Git Bash 提交有一定学习成本：
- 优点：完全掌握 Git 底层逻辑，适合深入学习，符合技术人员的专业习惯；
- 缺点：需要记忆命令、注意空格和语法，操作门槛高，容易出错，不适合快速提交。
### 2. 更方便、更有效率的提交方式
#### （1）GitHub Desktop
- 优点：全程鼠标点击，无需记忆命令，可视化查看修改、提交、推送，适合新手快速上手；
- 缺点：无法深入理解 Git 原理，过度依赖图形化操作，不利于技术成长。

#### （2）VS Code + Git 插件
- 优点：在代码编辑器中直接完成 Git 操作，结合代码编辑与版本控制，效率极高；
- 缺点：需要配置环境，对新手有一定学习成本。

#### （3）总结
Git Bash 是 Git 操作的基础，必须掌握；日常开发中可结合 VS Code 等工具提升效率，两者结合使用是最优方案。
