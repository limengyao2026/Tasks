# 核心问题回答
## 一、Geek与Geek精神
### 1. 什么是Geek
Geek（极客）是对技术有极致热爱、乐于探索、持续钻研的技术爱好者，不满足于“会用”，而是追求“懂原理、能创新、会分享”，是技术社区的核心贡献者。
### 2. 什么是Geek精神
Geek精神的核心是：
- **持续学习**：主动探索新技术，不满足于现有知识
- **开源共享**：乐于分享自己的学习成果、代码、笔记，帮助他人成长
- **解决问题**：遇到问题不放弃，通过搜索、实践、请教找到解决方案
- **追求极致**：对技术细节、文档规范、代码质量有高要求
- **创新精神**:勇于提出自己的想法与观点，敢于创造

## 二、GitHub核心概念理解
### 1. GitHub是什么
GitHub是全球最大的**代码托管与开源协作平台**，基于Git版本控制系统，用于：
- 个人代码/笔记托管、版本管理
- 团队协作开发、开源项目贡献
- 技术社区交流、学习资源分享
### 2. Fork & Pull Request 理解
- **Fork**：将他人的公开仓库完整复制一份到自己的GitHub账号，在副本上修改，不影响原仓库
- **Pull Request（PR）**：向原仓库提交修改申请，请求维护者合并自己的改动，是开源协作的核心流程
### 3. 学习来源
- GitHub官方文档：https://docs.github.com/zh

- 菜鸟教程GitHub入门：https://www.runoob.com/w3cnote/git-guide.html

## 三、GitHub访问经历与思考
### 1. 访问经历
本次访问GitHub时，遇到了**校园网访问超时（ERR_CONNECTION_TIMED_OUT）**、网页端创建仓库报错、找不到重命名按钮以及各种创建仓库的步骤等问题，通过以下方法解决：
- 网络问题：更换浏览器、使用国内镜像站、切换手机热点
- 操作问题：通过分步指引、绕开网页端限制（新建独立仓库替代重命名文件夹）解决
- 不熟练问题:询问豆包、deepseek等AI，以及查询浏览器，搜索相关教程等方法
### 2. 个人判断
GitHub作为境外平台，国内访问存在网络限制，需要掌握**镜像、Hosts、代理**等解决方法，才能顺利使用；同时GitHub网页端对文件夹操作有一定限制，需要用规范的仓库结构（独立Tasks仓库）来规避问题。

## 四、文档规范说明
### 1. 本文档遵循的规范
- **层级清晰**：使用`#`、`##`、`###`分级标题，逻辑清晰，结构一目了然
- **排版统一**：列表、段落、分隔线格式统一，无杂乱排版
- **链接可点**：所有外部链接均为可点击的超链接，方便查阅
- **内容完整**：覆盖所要求的内容，无遗漏
- **格式标准**：严格遵循Markdown语法，兼容所有Markdown编辑器
### 2. 规范的意义
规范的Markdown文档能让mentor快速查看作业完成情况，更清晰的看到框架逻辑，提升可读性和可维护性，同时培养自己良好的技术文档写作习惯，为后续开发工作打下基础。

# GitHub学习笔记
| 操作 | 作用 | 实操步骤 |
|------|------|----------|
| Fork | 复制仓库到自己账号 | 进入原仓库→点击右上角Fork→确认创建 |
| Commit | 提交修改到仓库 | 编辑文件→点击Commit changes→填写提交信息 |
| Create file | 创建新文件/文件夹 | 点击Add file→Create new file→输入`Tasks/README.md` |
| Sync fork | 同步原仓库更新 | 进入自己的仓库→点击Sync fork→Update branch |
| Pull Request | 提交修改到原仓库 | 点击Contribute→Open pull request→填写信息提交 |
